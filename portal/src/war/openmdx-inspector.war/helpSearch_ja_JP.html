詳細検索
詳細検索が使いやすくなりました
検索の際に演算子を使用することで、検索キーワードや数字をより細かく指定し、検索の性能をあげることができます。このページに記載されているオプションは全て検索ボックスに直接入力することが可能です。
基本操作
検索条件としては単語/数字または語句を入力することができます。例えば、'Miller', '> 500' や '>= Smith'のように入力し、
• column header をクリックして指定した条件にマッチするオブジェクトを検索します。(現在設定されているフィルタ条件にかかわらず全てのオブジェクトが検索対象になります。その際、既に取得済みの検索結果は失われます。) 新規検索
• [+] buttonをクリックして指定した条件にマッチするオブジェクトを検索します。(前回のフィルタ設定や検索結果に含まれるオブジェクトのみが検索対象となります。得られた検索結果からさらに条件を絞り込んで検索することができます。) 条件追加
演算子
サポートされる演算子は以下です。(※ データベースシステムによっては全ての演算子がサポートされない場合もあります。):
  • sounds like (*)
  • 部分検索 (%)、 文字列検索の規定値
  • 等しい (=)、ブール値、数値、コード検索の規定値
  • 等しくない (<>、!=)
  • より小さい (<)
  • 以下 (<=)
  • より大きい (>)
  • 以上 (>=)
"AND" 検索
一度の検索で複数の条件に同時にマッチするオブジェクトを得るには、 各条件の間にアンパサンド" &"を使用します。 例えば、500から1000までの数値(500より大きく1000より小さい)を含むオブジェクトを検索するには: '>500 & <1000'と入力します。
"OR" 検索
AまたはBという単語を含むオブジェクトを得るには各単語の間にセミコロン " ;" を使用します。 例えば、'Smith' または 'Miller'という名前の 担当者を検索するには: '= Smith; Miller'と入力します。
連結検索
複数の条件を実質上限なしに連結することができます。例えば、1000以下の、 500ではなく、かつ600ではない値を持つオブジェクトを検索するには: '<>500; 600 & <1000'と入力します。
連結検索の構文解析ルールは以下に基づいています:
  • searchExpr ::= filterValue { & filterValue }
    (--> string tokenizer with & as separator)
  • filterValue ::= [operator] value { ; value }
    (--> string tokenizer with ; as separator)
ワイルドカード検索

データベースによってはワイルドカードを使用できるものもあります。 例えば、'ller'を含む名前の担当者を全て検索するには: '%%ller'と入力します。
※データベースのチューニングによっては、ワイルドカード検索は非常に時間がかかる場合があります(全てのテーブルをスキャンするため)。
