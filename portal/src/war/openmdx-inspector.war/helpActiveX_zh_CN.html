ActiveX

基于ActiveX 控件特性的使用需求

  • 你的浏览器是 能够支持 ActiveX 控件的 (例如， MS Windows的Internet Explorer ) 并且
  • 安全设置是 权限允许引导/执行 ActiveX 控件

依赖于这个特性, 某些第三方软件包可能是需要的:

特性 需求 额外的提示
MS Word XML 合并 MS Word 2000 或更新版本基于客户端的安装 被信任的站点, ActiveX 权限
MS Excel XML 合并 MS Excel 2000 或更新版本基于客户端的安装  
XML 导出到 MS Outlook MS Outlook 2000 或更新版本基于客户端的安装  
XML Import from MS Outlook client-based installation of MS Outlook 2000 or newer  

被信任的站点 (Internet Explorer)

由于安全原因我们不建议你设置 ActiveX 控件 Internet 区域可用. 较好的做法是加入运行 openCRX 的站点到你的 被信任的站点区域 或者你的 局域网区域 (仅仅是站点确实在你的局域网中) 然后 使能 ActiveX 控件 在相应的区域. 下面是加入 openCRX demo 服务器的例子： 加入(demo.opencrx.org)到你的被信任站点区域然后加一个本地的 openCRX 服务器 (10.1.1.20)到局域网区域.
