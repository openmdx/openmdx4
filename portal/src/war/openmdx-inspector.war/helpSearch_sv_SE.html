Avancerad Sökning
Så här fungerar den avancerade sökfunktionen
För att öka träffsäkerheten i era sökningar, kan operatorer som finjusterar nyckelord och siffror användas. Samtliga sökfunktioner som presenteras på denna sida kan skrivas direkt i sökboxen.
Grundläggande funktioner
Ni kan skriva in ett ord/siffra eller ett uttryck, t.ex. 'Svensson', '> 500' eller '>= Andersson', och sedan
• klicka på en kolumns rubrik för att söka efter objekt som har matchande värde(n) för det attribut som rubriken beskriver(samtliga objekt genomsöks oberoende av preciserade sökfilter, och existerande sökresultat kommer förloras) Ny Sökning
• klicka på [+] knappen för att söka efter objekt med matchande värde(n) för just detta attribut (endast objekt som förekom i den föregående sökningen eller filtret kommer att sökas efter, dvs. att 'lägga till' denna sökning kommer att resultera i ett mer begränsat sökresultat) Lägg till villkor
Sökoperatorer
Följande sökoperatorer stöds (var god och beakta att vissa databassystem ej har stöd för samtliga nedanstående sökoperatorer):
  • låter som (*)
  • som (%), standard för söksträngar
  • lika med (=), standard för sökningar med Boolska (dvs. sant eller falskt), numeriska och kodvärden
  • ej lika med (<>, !=)
  • mindre än (<)
  • mindre än eller lika med (<=)
  • större än (>)
  • större än eller lika med (>=)
"OCH" Sökningar
För att finna objekt som matchar mer än ett villkor, används ett et-tecken "&" mellan villkoren. Till exempel, för att söka efter objekt med värden mellan 500 och 1000 (dvs. större än 500 och mindre än 1000), skriver ni följande i sökboxen: '>500 & <1000'
"ELLER" Sökningar
För att finna objekt, som inkluderar antingen ord A eller ord B, används ett semikolon ";" mellan orden. Till exempel, för att söka efter kontakter som heter 'Svensson' ELLER 'Andersson', skriver ni följande i sökboxen: '= Svensson; Andersson'
Kombinera Sökningar
Ni kan kombinera sökningar i princip utan begränsningar. Till exempel, för att söka efter objekt med värden mindre än '1000', men skilt från '500' och skilt från '600', skriver ni följande i sökboxen: '<>500; 600 & <1000'
Kombinerade sökningar används enligt följande regel:
  • searchExpr ::= filterValue { & filterValue }
    (--> strängarna separeras med hjälp av tecknet&)
  • filterValue ::= [operator] value { ; value }
    (--> strängarna separeras med hjälp av tecknet;)
Wildcard Sökningar

Vissa databaser tillåter att wildcard sökningar görs. Om detta stöds, kan ni exempelvis söka efter alla kontakter som innehåller 'son' (dvs. Andersson, Svensson, etc.) genom att skriva: '%%son'
VARNING: wildcard sökningar kan ta mycket lång tid om databasen ej är optimalt konfigurerad (full table scans)!
