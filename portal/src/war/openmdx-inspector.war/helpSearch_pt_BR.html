Pesquisa Avançada
Pesquisa Avançada de maneira fácil
Você pdoe incrementar a acuracidade de suas pesquisas adicionando operadores que refinam suas palavras-chaves e números. Todas as opções listadas nesta página pode ser entrado diretamente dentro da caixa de pesquisa.
As básicas
Você pode entrar com uma palavra/número ou uma expressão, e.g. 'Miller', '> 500' ou'>= Smith', e então
• clique no topo da coluna para pesquisar objetos com valores compatíveis a este atributo em particular (todos os objetos são pesquisados e lembrando que os filtros configurados anteriormente serão perdidos) Nova Pesquisa
• clique no botão [+] para pesquisar objetos com valores próximos a um atributo em particular (somente objetos contidos numa pesquisa anterior ou filtro são pesquisados, i.e. 'adicionando' esta pesquisa, irá retornar um conjunto/resultado mais restrito) Adicionar Condição
Operadores
Os seguintes operadores são suportados (entretanto, nem todos os banco de dados suportam todos os operadores):
  • parecido com (*)
  • igual (%), operador default pra pesquisa de strings
  • igual (=), operador default para boleano, número, e pesquisa de códigos
  • diferente (<>, !=)
  • menor que (<)
  • menor que ou igual (<=)
  • maior que (>)
  • maior que ou igual(>=)
Pesquisas com "AND"
Para recuperar objetos que atendam uma e outra ao mesmo tempo, use um E comercial "&" entre condições. Por exemplo, para pesquisar objetos com valores entre 500 e 1000 (i.e. maior que 500 e menor que 1000), simplesmente escreva: '>500 & <1000'
Pesquisas com "OR"
Para recuperar objetos que inclui a palavra A ou palavra B, use um ponto-e-vírgula ";" entre os termos. Por exemplo, para pesquisar por contatos com nomes 'Smith' OU 'Miller', simplesmente escreva: '= Smith; Miller'
Pesquisas combinadas
Você pode combinar pesquisas virtualmente sem limites. Por exemplo, para pesquisar objetos com valores menor que '1000', mas diferente de '500' e diferente de '600', simplesmente escreva: '<>500; 600 & <1000'
As regras de parsing para pesquisas combinadas são as que seguem:
  • expressãoDePesquisa ::= valorDoFiltro { & valorDoFiltro }
    (--> divisão de strings com & como separador)
  • valorDoFiltro ::= [operador] valor { ; valor }
    (--> divisão de strings com ; como separador)
Pesquisas por 'curingas'

Alguns bancos de dados permitem você iniciar pesquisas usando 'curingas' . Se suportado, você pode por exemplo procurar por todos os contatos com nomes que contenham 'ller' (e.g. Miller, Mueller, etc.) escrevendo: '%ller'
ATENÇÃO: pesquisas com 'curingas' pode consumir muito tempo do banco de dados se ele não estiver bem ajustado (pesquisa completa da tabela)!
